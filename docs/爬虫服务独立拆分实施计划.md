# 爬虫服务独立拆分实施计划

## 目标

将 `MediaCrawler-Api` 拆分为两个纯 API 通信的微服务，**零代码共享**，仅通过 HTTP API 契约通信。

| 服务 | 端口 | 职责 | 数据库 |
|------|------|------|--------|
| Crawler Service | 8081 | 爬虫控制 + WebSocket + 爬取执行 | 无 |
| Data API Service | 8080 | 数据持久化/查询/AI/配置/平台管理 | MySQL |

## 两个服务各保留的模块

### Crawler Service (:8081)
- `media_platform/` (7平台爬虫实现)
- `base/` (抽象基类)
- `store/` → 重写为 HTTP store
- `model/` (平台 Pydantic 数据模型，序列化用)
- `config/` (爬虫运行时参数)
- `tools/` (CDP浏览器、文件写入、工具)
- `proxy/` (IP代理)
- `cache/` (本地缓存)
- `var.py` (上下文变量)
- `cmd_arg/` (CLI参数解析)
- `main.py` (CLI入口)
- `api/main.py` (FastAPI app)
- `api/routers/crawler.py` (爬虫控制 REST)
- `api/routers/websocket.py` (实时日志/状态)
- `api/services/crawler_manager.py` (子进程管理)
- `api/schemas/crawler.py` (API 契约)

### Data API Service (:8080)
- `api/main.py` (FastAPI app)
- `api/routers/data.py` (文件数据)
- `api/routers/data_db.py` (MySQL 数据查询)
- `api/routers/ai.py` (AI 对话 + 评论分析)
- `api/routers/keywords.py` (关键词管理)
- `api/routers/config_mgmt.py` (配置方案 CRUD)
- `api/routers/platforms.py` (平台管理)
- `api/routers/system.py` (系统状态)
- `api/routers/internal.py` (NEW — 供 Crawler 调用的内部 API)
- `api/schemas/crawler.py` (API 契约，序列化/反序列化用)
- `database/` (ORM 模型)
- `services/config_service.py` (profile CRUD + task CRUD)
- `services/data_query_service.py` (数据查询)
- `services/platform_service.py` (平台管理)
- `tools/utils.py` (logger)
- `config/db_config.py` (数据库配置)
- `cache/` (缓存)

### 删除的模块
- `MediaCrawler-Api/` 原目录 → 重命名为 `crawler-service/` 后删减

## 服务间 API 契约

### Data API Service 新增内部接口

```
POST /api/internal/data/batch
Body: {
  "platform": "xhs",
  "kind": "content|comment|creator",
  "task_id": 65,
  "records": [{...}, {...}]
}

GET  /api/internal/tasks/{id}
Response: { task_id, status, payload: {...} }

PUT  /api/internal/tasks/{id}/progress
Body: { progress: 45, message: "已采集 45/100" }

PUT  /api/internal/tasks/{id}/finish
Body: { status: "completed|failed", error: "..." }
```

### Crawler Service 启动爬虫流程

```
1. 前端 POST /api/crawler/start (带 payload) → Crawler Service
2. CrawlerManager 先调 Data API: POST /api/internal/data/batch (创建 task)
3. CrawlerManager 启动 subprocess: uv run python main.py --task-id N --data-api-url http://127.0.0.1:8080
4. subprocess 调 Data API: GET /api/internal/tasks/{id} 获取配置
5. 爬虫执行，store 改为调 Data API: POST /api/internal/data/batch 写入数据
6. progress_reporter 改调 Data API: PUT /api/internal/tasks/{id}/progress
7. 完成时调 Data API: PUT /api/internal/tasks/{id}/finish
```

## 实施任务

### 任务 1：创建 Crawler Service 项目骨架
- 将 `MediaCrawler-Api/` 复制为 `crawler-service/`
- 删除不需要的模块（database/, services/config_service.py, services/data_query_service.py, services/platform_service.py, api/routers/data.py, data_db.py, ai.py, keywords.py, config_mgmt.py, platforms.py, system.py, api/webui/）
- 创建简化的 pyproject.toml

### 任务 2：创建 Data API Service
- 从 MediaCrawler-Api 复制数据 API 相关模块到 Data-API-Service/
- 删除爬虫相关模块（media_platform/, base/, store/, proxy/, model/, constant/, cmd_arg/, main.py, var.py, tools/除 utils.py）
- 新增 api/routers/internal.py（内部 API）

### 任务 3：改造 Crawler Service 的 Store 层为 HTTP Store
- 创建通用的 HttpStore 抽象基类
- 改造 7 个平台的 _store_impl.py 从 SQLAlchemy → HTTP POST
- 改造 excel_store_base.py

### 任务 4：改造 Crawler Service 的 task_loader 和 progress_reporter
- task_loader.py → HTTP GET 获取配置
- progress_reporter.py → HTTP PUT 上报进度
- crawler_manager.py → HTTP 创建任务 + 标记完成

### 任务 5：前端 Vite 代理 + monorepo 脚本更新
- vite.config.ts 分拆代理
- package.json 更新启动脚本
- pnpm-workspace.yaml 更新

### 任务 6：依赖安装 + 集成测试
- 两个服务各自 uv sync
- 启动 MySQL → Data API → Crawler → 前端
- 端到端测试爬虫流程
