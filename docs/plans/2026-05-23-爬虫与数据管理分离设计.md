# 爬虫能力与数据管理分离 — 架构设计方案

日期：2026-05-23
状态：已确认，待实施

---

## 一、背景与目标

当前 MediaCrawler-Api 是单体结构，爬虫引擎、数据查询、AI 对话、关键词管理等全部耦合在一个 FastAPI 服务中。需要拆分为独立服务，实现：

- **独立服务部署**：Crawler Service 和 Data API Service 各自独立运行、独立端口
- **微服务化预留**：目录结构支持未来进一步拆分（如 AI Service 独立）

## 二、拆分决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 拆分粒度 | 双服务（Crawler + Data API） | 当前最小可行拆分，AI/关键词后续可独立 |
| 数据库 | 共享同一 MySQL 实例/同一库 | 避免数据迁移，简化查询 |
| 服务间通信 | REST API | 低延迟、易调试 |
| 功能归属 | AI + 关键词 + 平台 → Data API | 它们是数据管理的附属能力 |
| 前端对接 | 直连多服务 | 无需 API Gateway，前端配置多 baseURL |

## 五、数据库表使用分配

| 分类 | 表 | 写入方 | 读取方 |
|------|-----|--------|--------|
| 业务数据(18张) | xhs_note, douyin_aweme, bilibili_video 等 | Crawler Service (Store层) | Data API Service (DataQuery) |
| 爬虫管理(2张) | crawler_profile, crawler_task | Crawler Service | 双方 |
| 数据管理(3张) | keyword, keyword_group, platform | Data API Service | Data API Service |
| AI(2张) | chat_session, chat_memory | Data API Service | Data API Service |

## 六、Crawler Service REST API 契约

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/crawler/start` | 启动爬虫任务 |
| POST | `/api/crawler/stop` | 停止当前爬虫 |
| GET | `/api/crawler/status` | 查询运行状态 |
| GET | `/api/crawler/tasks` | 分页查询历史任务 |
| GET | `/api/crawler/tasks/{id}` | 单任务详情 |
| POST | `/api/crawler/tasks/{id}/rerun` | 重新执行 |
| DELETE | `/api/crawler/tasks/{id}` | 删除任务 |
| GET | `/api/crawler/logs` | 获取日志 |
| POST | `/api/crawler/cleanup-zombies` | 清理僵尸进程 |
| WS | `/api/ws/logs` | WebSocket 实时日志 |
| WS | `/api/ws/status` | WebSocket 状态推送 |

## 七、前端改动

1. `.env` 新增 `VITE_CRAWLER_BASE_URL=http://localhost:8081`
2. `api/request.ts` 支持多 baseURL：匹配 `/crawler` 和 `/ws` 的请求走 Crawler Service，其余走 Data API Service
3. `vite.config.ts` 新增 Crawler Service 代理规则
4. `useCrawlerLogs.ts` WebSocket URL 从环境变量读取

## 八、实施顺序

| 阶段 | 内容 | 验收标准 |
|------|------|----------|
| 1 | 创建 shared/ 共享包，提取 ORM 模型和 db_session | shared 包可被 pip install |
| 2 | 创建 crawler-service/，搬移爬虫相关代码，修改 import | 爬虫 CLI 能独立运行 |
| 3 | 精简 data-api-service/，移除爬虫代码，引入 shared | Data API 能独立运行 |
| 4 | Crawler Service 添加 REST API 层 | `/api/crawler/start` 可通过 HTTP 调用 |
| 5 | 前端适配多服务配置 | 前端能同时访问两个服务 |
| 6 | 集成测试 + 端到端验证 | 完整爬虫→数据查询链路通 |
| 7 | 清理旧文件 | 无残留代码 |
